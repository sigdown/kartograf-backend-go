## Введение

Перед публикацией карты нужно выполнить два шага:

- геопривязать исходные листы
- собрать итоговый `.pmtiles`-архив

Геопривязка - это привязка изображения к мировым координатам по набору опорных точек.

Подготовка архива - это сборка геопривязанных изображений в единый `.pmtiles`-файл, пригодный для дальнейшей публикации и раздачи тайлов.

## Инструменты

В текущем процессе используются:

- `QGIS` - геопривязка и ручная проверка результата
- `gdal` - reprojection, warp, сборка мозаики, подрезка
- `rio-pmtiles` - упаковка GeoTIFF в `.pmtiles`

## Геопривязка

Геопривязка исходных листов выполняется в `QGIS`.

### Шаг 1. Оценить качество исходника

Перед началом нужно понять, подходит ли изображение для геопривязки.

Проблемные случаи:

- сильно растянутое изображение
- выраженная деформация
- сильный поворот
- повреждённые или обрезанные края

Чем хуже исходник, тем сложнее получить хороший результат.

### Шаг 2. Выбрать тип трансформации

Базовые варианты:

- `Линейный` - слабая деформация или её отсутствие, минимум 2 точки
- `Полиномиальный 1` - есть поворот, минимум 3 точки
- `Полиномиальный 2` - есть слабые нелинейные деформации, минимум 6 точек
- `Полиномиальный 3` - сильные нелинейные деформации, минимум 10 точек
- `Тонкостенный сплайн` - локальная подгонка, используется при сложной деформации и хорошем наборе контрольных точек

### Шаг 3. Получить геопривязанные листы

На выходе должен получиться набор файлов вида:

```text
sheet_01_georef.tif
sheet_02_georef.tif
sheet_03_georef.tif
...
```

## Подготовка master TIFF

### Шаг 1. Привести листы к Web Mercator

```powershell
gdalwarp -t_srs EPSG:3857 -r lanczos -dstalpha sheet_01_georef.tif sheet_01_3857.tif
gdalwarp -t_srs EPSG:3857 -r lanczos -dstalpha sheet_02_georef.tif sheet_02_3857.tif
```

### Шаг 2. Собрать виртуальную мозаику

```powershell
gdalbuildvrt mosaic.vrt sheet_*_3857.tif
```

### Шаг 3. Проверить мозаику в QGIS

Нужно проверить:

- швы между листами
- стыковку рек, дорог и застройки
- общую корректность положения
- необходимость подрезки рамок

### Шаг 4. Собрать итоговый GeoTIFF

```powershell
gdalwarp -multi -wo NUM_THREADS=ALL_CPUS mosaic.vrt mosaic_3857.tif
```

### Шаг 5. При необходимости подрезать края

```powershell
gdalwarp `
  -cutline neatline.geojson `
  -crop_to_cutline `
  -dstalpha `
  mosaic_3857.tif mosaic_3857_cut.tif
```

## Сборка PMTiles

Установить инструменты:

```powershell
pip install rio-pmtiles pmtiles
```

Собрать архив:

```powershell
rio pmtiles mosaic_3857.tif map.pmtiles `
  --format WEBP `
  --rgba `
  --tile-size 512 `
  --exclude-empty-tiles `
  --co QUALITY=85
```

Проверить результат:

```powershell
pmtiles verify map.pmtiles
```

## Итог

На выходе должен получиться готовый `.pmtiles`-архив, который можно загрузить в object storage и опубликовать в системе.
